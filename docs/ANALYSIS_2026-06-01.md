## The journey (what moved the needle)
1. **Supervised (behavior cloning)** on the 98k-game official dataset → legal but weak.
2. **PPO vs a frozen SL baseline** → first model to beat SL (`ppo_vb`, +432). Pure self-play
   *failed* (co-evolves an exploitable aggressive meta); training vs a frozen opponent worked.
3. **Opponent pool (league)** → `poolbig` beat `ppo_vb` by training against a *pool* of our own
   strong bots. **Key finding: an opponent pool broke the plateau; a 4× larger batch did not.**
4. **Population-Based Training (PBT)** across the fleet → gen-2 champion, another increment.

## Two hard, honest findings
- **Plateau.** Self-play draws ~88–90% of games (4 cautious strong bots rarely feed each other
  8-fan wins). Improvements are real but shrinking; the *conversion* problem (building high-fan
  hands) is the deep, unsolved issue.
- **Coevolution drift.** In the PBT run the population *peaked at gen 2* then wandered — gen 8
  ended up *worse than poolbig*. The per-generation metric is noisy; only a **unified cross-play
  tournament** ranks reliably. The rolling-deploy file had been overwritten to the regressed
  gen 8 — caught before shipping. **Fix applied: a global-best hall of fame** — the deployed
  model is only ever replaced by a candidate that *beats it* in a held-out tournament.

## RESULT of the 7.7h unattended run (`train/improve_local.py`) — a negative result
**The run drifted; we rolled the deploy back to gen2.** 28 rounds, 12 "promotions". But a
clean post-hoc tournament (250 games/pair, then 700/pair to confirm) ranked the models:

    gen2 +3360  >  deploy_r25 +142  >  poolbig -1050  >  imp_r2 -2452   (700 games/pair)

The loop deployed **r25, which is *worse* than the gen2 champion it started from.** Cause:
the per-round selection tournament was only 240 games over a fixed 4-way — noisy enough that a
candidate can beat the reigning best *by chance*, get promoted, and degrade the reference for
every subsequent round (classic coevolution drift / noisy-selection failure). The hall-of-fame
logic was sound (valid head-to-head vs current best) but **240 games is too few to be the
selector.** Bonus finding: `imp_r2` scored well in-pool but **collapsed when the field changed**
(+3374 in the 7-way → −2452 in the top-4) — the improve champions are **overfit to our own
pool** and don't generalize.

**Action taken:** restored `pbt_champion_fp16.npz` ← `pbt_gen2_champion_fp16.npz` (md5-verified).
Final deploy-format confirmation: DEPLOY(gen2 fp16) **+5258** > drifted_r25 +3578 > poolbig +3250.
Drifted file kept as `pbt_champion_fp16.DRIFTED_r25.npz.bak`.

**Lessons for the next run (mechanism, not just hyperparams):**
1. **Select against a *fixed* anchor (gen2), not a drifting "best".** Promote only if cand beats
   the frozen gen2 anchor *and* the current best — never let the reference degrade.
2. **≥600–800 games/pair for any promotion decision.** 240 is noise.
3. **Hold out a generalization check**: score candidates against models *not* in their training
   pool before promoting (catches pool-overfit like imp_r2).
4. One real signal did survive: promotions clustered at moderate LR (8e-6–1e-5) with non-zero
   fan-potential shaping (0.02–0.03); `shape=0.0` rounds reliably failed — shaping is doing work.

## What the 8h unattended run did (`train/improve_local.py`)
The vast fleet is contended (often only 2–3 free boxes, fluctuating), so the unattended loop
runs on the reliable **local 28-core box**. Each round:
1. train one PPO candidate from the current **global best** vs the growing pool, with sampled
   hyperparameters (lr / shape / entropy / games / iters);
2. run a held-out cross-play tournament: candidate vs global-best vs frozen anchors
   (`poolbig`, `bc_v3_ft`);
3. **promote only if it wins** → updates `pbt_champion_fp16.npz` and joins the pool.
- Live status: `/tmp/improve_status.json`; log in the task output. Reports `running/ping/finished`
  to the central fleet dashboard. The deployed champion **cannot regress** by design.

## The binding constraint (most important)
The single highest-value input is a **real Sim-7 ladder log** vs the other competitors. All our
numbers are self-play/in-house tournaments, which can't measure conversion against *diverse*
opponents and may flatter models that overfit our own pool. A few ladder games would let us add
real opponents to the pool — worth more than any amount of additional self-play.

## Open directions (post-run)
1. **Attack conversion directly**: reward shaping for *fan potential* (not just shanten), and
   value-aware claim decisions — the lever for the 88%-draw ceiling.
2. **Feed Sim-7 ladder opponents into the pool** once a log is available.
3. **Anti-drift league**: prioritized fictitious self-play + permanent old champions in the pool.
4. Resume/extend PBT across the fleet when more boxes free up (`train/pbt_loop.py`, hall-of-fame
   already added).

Trust that, not multi-way net, for "is A better than B".

## 2026-06-02: planning + value-net dev — honest negative, and the recurring wall
Pursued the test-time PLANNING direction (use the 6s budget; we use ~0.3s) + value net.
- **Value head is useless for search:** predictions collapsed to ~0.056±0.027 (the mean, in an
  88%-draw domain), corr with outcome 0.16. No 1-ply value search.
- **Naive fan-potential rollout planner (train/plan.py) HURTS.** Position-controlled paired A/B
  (run each seed with planner vs greedy in the SAME seats 0,2 — cancels seat bias): planning
  effect **-217 to -284**, draws MORE (87-91% vs 84%), wins LESS (2-4% vs 7%); robust across
  horizons. Cause: a SOLITAIRE rollout (free draws, ignores opponents winning first) overvalues
  big-hand fan-chasing; the RL policy already balances speed/fan/defense better. (Also caught a
  phantom +830 "win" that was pure seat-0,2 positional bias — the all-greedy control reproduced
  it exactly. Lesson: always position-control paired A/Bs.)
- **The deeper pattern:** every in-house lever (bigger batch, PBT, league, sampled-pool, value
  net, planning) converges to the same passive, draw-heavy, near-gen2 equilibrium — because it is
  optimal *vs itself*. Whether more aggression/conversion helps *against the field* is unmeasurable
  in-house. The bot's passivity (≈4 claims/game on the real judge) is optimal self-play, maybe not
  optimal vs real opponents. **The wall is the same as always: we need a Sim-7 ladder log.**
- Ladder-independent dev that IS still worth doing: (a) HL regression-suite infra (so we iterate
  fast + safely when ladder data lands); (b) DEFENSE / deal-in avoidance (measurable vs aggressive
  in-house opponents, robustly valuable vs any field). Proper opponent-aware PIMC is deferred
  (large, uncertain vs a strong RL policy).

## 2026-06-02 (session 2): defense is a NON-lever; in-house model search is exhausted
Investigated the two ladder-independent levers above with data. Both turned out to be dead ends —
and the investigation produced the cleanest statement yet of the wall.

**Drift caught & contained.** A *second*, unsupervised league run (round 7, champion r5) had died
~3h prior and silently overwritten the deploy file (`pbt_champion_fp16.npz`) with an unverified
fp16 (`8ee6b41…`). Restored deploy to the supervised r18 (`league_v3_champion_fp16.npz`,
`4d465a9…`); backup of the artifact kept as `pbt_champion_fp16.OVERWRITTEN_r7run_8ee6b414.npz`.

**The "+4008" r18-vs-gen2 margin was a harness artifact.** With one consistent in-process harness
(`eval/h2h_fast.py`): r18 vs gen2 = **+142** (1200g, 90% draw); r7run vs gen2 = −268; r7run vs r18 =
+864. All in the few-hundred range → **in-house self-play margins under ~±400/1200g are NOISE.**
gen2 ≈ r18 ≈ r7run ≈ poolbig — all contest-equivalent.

**Defense / deal-in avoidance is a NON-lever (data, `eval/outcome_stats.py`).** Champion deal-in
rate: **1.8% self-play, 0.7% vs a weaker/looser bot.** The bot already plays near-optimally
defensively; max recoverable points from eliminating deal-ins (~1% of games × ~20pts) is far below
the ±400 noise floor. The problem was never defense — it's purely CONVERSION (89% draws).

**Conversion is only measurable vs a fixed weak field — and that metric is opponent-specific
NOISE.** New idea: rank models by farming-net vs a fixed weak bot (contest-relevant: you score by
beating the weak members of a mixed field). gen2 led vs bc_v3_ft (+11760 vs r18 +10014) — but the
lead REVERSED vs ppo_vb (+494 vs +1698) and bc_tiny (+9100 vs +9587). The farming "winner" is
opponent-specific → training a model to a fixed-pool farming fitness would overfit that pool (the
`imp_r2` failure). So the farming-fitness training run was NOT launched.

**Verdict: in-house model search is exhausted.** Every model we have is contest-equivalent within
noise; both ladder-independent strength levers (defense, farming-fitness) are dead. No further
self-play/league/farming run will reliably produce a better contest model. The binding constraint
is now the ONLY lever: a real Sim-7 / external-opponent ladder log.

**Built (ladder-ready scaffolding):**
- `eval/h2h_fast.py` — fast position-controlled in-process head-to-head (the trustworthy A>B test).
- `eval/outcome_stats.py` — classifies game outcomes from score vectors (deal-in / self-draw /
  rong / draw rates) for any policy pair.
- `eval/replay_harness.py` — reconstructs a Botzone match log into per-seat decision points
  (draw-turns + claim-chances) for regression testing / failure mining. On the real 4-bpt log it
  confirms **only 3% of claim chances are acted on** (extreme passivity on the real judge).
  Next increment: drive `bot/ml_bot.py`'s full decision logic (not raw net argmax) for faithful
  per-action reproduction.
